## 读写路由规则

### 优先级（从高到低）

1. **方法上的 `@DS` 注解** — 最高优先级，显式声明
2. **类上的 `@DS` 注解** — 对整个类生效
3. **方法名约定** — 以 `get/find/list/query/count/select/fetch/load/search` 开头 → SLAVE
4. **兜底** → MASTER（保证写安全）

### 各 Service 方法路由汇总

| Service | 方法 | 路由 | 原因 |
|---------|------|------|------|
| UserService | `register()` | **MASTER** | 写操作，INSERT |
| UserService | `login()` | **SLAVE** | 读操作，SELECT |
| ProductService | `listOnSale()` | **SLAVE** | 读操作，SELECT |
| ProductService | `getById()` | **SLAVE** | 读操作，SELECT |
| OrderService | `placeOrder()` | **MASTER** | 写操作，事务 |
| OrderService | `payOrder()` | **MASTER** | 写操作，UPDATE |
| OrderService | `cancelOrder()` | **MASTER** | 写操作，UPDATE |
| OrderService | `getMyOrders()` | **SLAVE** | 读操作，SELECT |
| OrderService | `getByOrderNo()` | **SLAVE** | 读操作，SELECT |
| SeckillService | `doSeckill()` | **MASTER** | 写操作，事务 |
| SeckillProductService | `listAll()` | **SLAVE** | 读操作，SELECT |
| SeckillProductService | `getById()` | **SLAVE** | 读操作，SELECT |
| SeckillProductService | `warmUpAllStock()` | **MASTER** | 预热读主库，避免主从延迟 |

## 快速启动

### 1. 启动所有服务

```bash
docker-compose up -d
```

### 2. 等待主库健康（约 30~40s），然后配置主从复制

```bash
docker-compose exec mysql-slave bash /docker-entrypoint-initdb.d/setup-replication.sh
```

### 3. 验证主从复制状态

```bash
docker-compose exec mysql-slave mysql -uroot -proot123 \
  -e "SHOW SLAVE STATUS\G" | grep -E "Running|Behind"
# 期望输出：
#   Slave_IO_Running: Yes
#   Slave_SQL_Running: Yes
#   Seconds_Behind_Master: 0
```

## 注意事项

### 主从延迟
- **注册后立即登录**：`register()` 写主库，若主从有延迟，`login()` 走从库可能查不到新用户。当前方案：`register()` 返回 UserVO 含 token，无需立即查从库。
- **秒杀预热**：`warmUpAllStock()` 强制走主库，避免从库未同步时预热到旧库存。
- 如需强一致，可在特定 Service 方法上标注 `@DS(MASTER)` 覆盖约定。

### 事务内不切库
- `DataSourceAspect` 的 `@Order(1)` 确保在事务开始前切库，同一事务内所有 SQL 使用同一连接，事务不会失效。

### 从库只读保护
- `mysql/slave/my.cnf` 配置了 `read-only=ON` 和 `super-read-only=ON`，从库拒绝任何写操作，即使误路由也会报错而非静默写入。
